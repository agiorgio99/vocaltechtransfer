# VocalTechTransfer

Exploration of vocal technique transfer in EnCodec latent space, using the GTSinger dataset (EN-Tenor-1).

**Research question:** Does EnCodec latent space linearly encode vocal techniques? Can we transfer a technique from one performance to another by interpolating latents, and at what α does the transfer become perceptually convincing?

**Method:** Latent interpolation inspired by Tokui & Baker (2025) — encode two paired recordings (Control and Technique group) with EnCodec, interpolate their latents, decode back to audio.

---

## Dataset

**GTSinger EN-Tenor-1** — a multi-technique singing dataset with paired recordings:
- `Control_Group` — natural singing, no technique applied
- `[Technique]_Group` — same melody with the technique densely applied

Techniques: Vibrato, Breathy, Glissando, Mixed Voice & Falsetto, Pharyngeal.

Each pair shares the same lyrics and melody, making it a controlled transfer experiment.

The `data/normalized/` folder contains WAVs already resampled to 24 kHz mono (required by EnCodec). File naming convention:

```
{Technique}__{song_name}__{Group}__{phrase_index}.wav
```

### Data setup

The `data/` folder is not included in this repository. To reconstruct it:

1. Download the **English** zip from the GTSinger Google Drive:  
   <https://drive.google.com/drive/folders/1xcdvCxNAEEfJElt7sEP-xT8dMKxn1_Lz?usp=drive_link>

2. Extract the archive and place the `EN-Tenor-1` folder inside a `GTSinger` subfolder under `data/`:

```
data/
└── GTSinger/
    └── EN-Tenor-1/
        ├── Vibrato/
        ├── Breathy/
        ├── Glissando/
        ├── Mixed_Voice_and_Falsetto/
        └── Pharyngeal/
```

3. Run **Sections 1 and 2** of the notebook (`VocalTechTransfer.ipynb`) once to extract and normalise all WAV files into `data/raw_wav/` and `data/normalized/`. Everything from Section 3 onwards can then be run freely.

## Audio samples

After running the notebook, all interpolated audio files are written to the `outputs/` directory:

```
outputs/
├── encodec/
│   └── audio/    ← EnCodec 24 kHz interpolations
└── dac/
    └── audio/    ← DAC 44.1 kHz interpolations
```

**File naming convention:**

```
{Technique}__{Song}__{Phrase}__a{alpha*100:03d}.wav
```

For example, `Vibrato__Rolling_in_the_Deep__0000__a075.wav` is the Vibrato transfer at α = 0.75.

**Listening to the samples** — open any WAV with your preferred audio player, or load them inline in the notebook via the `IPython.display.Audio` widgets already present in Sections 5, 8, 10b, 10e, and 11.

The five α levels produced for each (technique, song, phrase) pair are:

| File suffix | α | What you hear |
|-------------|---|---------------|
| `a000` | 0.00 | Clean control recording; no audible technique |
| `a025` | 0.25 | Subtle timbral shift; technique barely perceptible |
| `a050` | 0.50 | Noticeable blend; technique starts to emerge |
| `a075` | 0.75 | Technique clearly audible; mild reconstruction artefacts |
| `a100` | 1.00 | Full technique performance; codec baseline quality |

The `outputs/encodec/figures/` and `outputs/dac/figures/` subdirectories contain all PCA, Δz, and orthogonality plots generated by the notebook.

## Notebook outline

| Section | What it does |
|---------|-------------|
| **0. Setup** | Imports, path definitions, technique/colour maps |
| **1. Extract WAVs** | Copy GTSinger WAVs into `data/raw_wav/` with flat naming |
| **2. Normalise audio** | Resample to 24 kHz (EnCodec) and 44.1 kHz (DAC) |
| **3. Load EnCodec** | Load `facebook/encodec_24khz`, build token→latent lookup table |
| **4. Encode a matched pair** | Encode Control + Technique recordings into latent space |
| **5. Interpolation sweep** | α = 0 → 1, decode and play each step (EnCodec) |
| **6. Latent geometry** | Distributions, L2 norm over time, PCA trajectory, Δz bar charts |
| **7. All techniques** | Comparative PCA, Δz heatmap, orthogonality matrix (EnCodec) |
| **8. Transfer demo** | Multi-song/phrase transfer at α = 0.75 (EnCodec) |
| **9. EnCodec summary** | Per-technique Δz magnitudes and top dimensions |
| **10. DAC** | Full repeat of pipeline with Descript Audio Codec at 44.1 kHz |
| **11. EnCodec vs DAC** | Side-by-side waveforms, PCA, orthogonality bar chart |
| **12. Final summary** | Consolidated results table |
